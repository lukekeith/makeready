**1. Create `resources/js/components/admin/member-filter-bar/member-filter-bar.vue`:**

This is a presentational component — NO direct store access. All data via props, all mutations via emits.

Props:
- `filterTags: FilterTag[]` — current active filter chips
- `searchQuery: string` — current search input value
- `availableGroups: string[]` — group names for combobox suggestions
- `hasActiveFilters: boolean` — whether to show clear-all button

Emits:
- `(e: 'add-filter', category: FilterTag['category'], value: string): void`
- `(e: 'remove-filter', index: number): void`
- `(e: 'clear-filters'): void`
- `(e: 'update:search-query', value: string): void`

Import from reka-ui: `ComboboxRoot`, `ComboboxInput`, `ComboboxContent`, `ComboboxItem`, `ComboboxEmpty`.
Import `X`, `Search` from `lucide-vue-next`.

Template structure:
```
div.MemberFilterBar
  div.MemberFilterBar__search-row
    div.MemberFilterBar__search-wrapper
      Search icon (16px, muted color, absolute positioned left)
      input.MemberFilterBar__search-input
        v-model bound to local ref that emits update:search-query
        placeholder="Search members..."
        @keydown.enter.prevent → emit('add-filter', 'name', searchQuery) if non-empty
      button.MemberFilterBar__add-btn @click → emit('add-filter', 'name', searchQuery)
        text: "Add"

  div.MemberFilterBar__dropdowns
    <!-- Group combobox -->
    ComboboxRoot(@update:model-value="(v) => emit('add-filter', 'group', v)")
      ComboboxInput(placeholder="Group..." class="MemberFilterBar__combobox-input")
      ComboboxContent(class="MemberFilterBar__combobox-content")
        ComboboxEmpty(class="MemberFilterBar__combobox-empty") "No groups found"
        ComboboxItem(v-for="group in availableGroups" :value="group" class="MemberFilterBar__combobox-item")
          {{ group }}

    <!-- Status combobox -->
    ComboboxRoot(@update:model-value="(v) => emit('add-filter', 'status', v)")
      ComboboxInput(placeholder="Status..." class="MemberFilterBar__combobox-input")
      ComboboxContent(class="MemberFilterBar__combobox-content")
        ComboboxItem(v-for="status in ['Completed', 'In Progress', 'Upcoming']" :value="status" class="MemberFilterBar__combobox-item")
          {{ status }}

    <!-- Activity type combobox -->
    ComboboxRoot(@update:model-value="(v) => emit('add-filter', 'type', v)")
      ComboboxInput(placeholder="Activity type..." class="MemberFilterBar__combobox-input")
      ComboboxContent(class="MemberFilterBar__combobox-content")
        ComboboxItem(v-for="type in ['SOAP', 'VIDEO', 'READ', 'OIA', 'DBS', 'HEAR', 'USER_INPUT']" :value="type" class="MemberFilterBar__combobox-item")
          {{ type }}

  <!-- Active filter chips -->
  div.MemberFilterBar__chips(v-if="hasActiveFilters")
    div.MemberFilterBar__chip(
      v-for="(tag, index) in filterTags"
      :key="index"
      :class="'MemberFilterBar__chip--' + tag.category"
    )
      span.MemberFilterBar__chip-label {{ formatLabel(tag) }}
      button.MemberFilterBar__chip-remove(@click="emit('remove-filter', index)")
        X(:size="12")
    button.MemberFilterBar__clear-all(@click="emit('clear-filters')")
      "Clear all"
```

Helper function `formatLabel(tag: FilterTag): string`:
- Map category to display name: name→"Name", group→"Group", status→"Status", type→"Type"
- Return `"${label}: ${tag.value}"`

Use a local `ref` for the search input that syncs with the prop via `watch` and emits `update:search-query` on change. When enter or Add is clicked, emit `add-filter` with `('name', localSearch.value)` and reset the local ref.

            'admin_user_session' => 'test-session-id',

    }
