**1. Create `resources/js/islands/admin-island/stores/ui/members-list.ui.ts`:**

Define and export `FilterTag` interface with `category: 'name' | 'group' | 'status' | 'type'` and `value: string`.

Create Pinia store `useMembersListUI` following the established `defineStore('members-list-ui', () => {})` pattern (reference `groups-list.ui.ts` for structure).

Inside the store:
- Call `useAllMembersDomain()` to get the domain store reference
- `filterTags = ref<FilterTag[]>([])`
- `searchQuery = ref('')`
- `dismissedFailedGroups = ref(false)`
- `filteredMembers = computed<UnifiedMember[]>(() => { ... })` — if `filterTags.value.length === 0`, return `domain.allMembers`. Otherwise, filter with `.every()` applying AND logic:
  - `'name'`: `member.name.toLowerCase().includes(tag.value.toLowerCase())`
  - `'group'`: `member.groups.some(g => g.groupName.toLowerCase() === tag.value.toLowerCase())`
  - `'status'` and `'type'`: `return true` (stub — Phase 14 activity data not yet available). Add code comment: `// TODO(Phase 14): Filter by activity data once available`
- `hasActiveFilters = computed(() => filterTags.value.length > 0)`
- `addFilter(category, value)`: trim check, duplicate prevention (same category+value), push to filterTags, reset searchQuery
- `removeFilter(index)`: splice at index
- `clearFilters()`: reset filterTags to [], searchQuery to ''
- `setSearchQuery(query)`: set searchQuery
- `dismissFailedGroups()`: set dismissedFailedGroups to true

Export `memberColumns` — an array of `ColumnDef<UnifiedMember>[]` defined using `createColumnHelper<UnifiedMember>()` from `@tanstack/vue-table`:
  1. `display({ id: 'avatar' })` — renders `<img>` with `src: row.original.avatarUrl`, class `AdminVirtualTable__avatar`, 32px circle. If no avatarUrl, render a div with the member's initials (first letter of name) with class `AdminVirtualTable__avatar-placeholder`.
  2. `accessor('name', { header: 'Name' })` — renders name text directly.
  3. `display({ id: 'groups', header: 'Groups' })` — renders a `<div>` with class `AdminVirtualTable__group-chips` containing `<span>` elements with class `AdminVirtualTable__group-chip` for each group in `row.original.groups`, showing `g.groupName`.
  4. `accessor('lastActive', { id: 'lastActive', header: 'Last Active' })` — renders relative time using a `formatRelativeTime(dateStr)` helper function. Helper: parse ISO string, compute diff from now, return "Xd ago", "Xh ago", "Xm ago", or "just now". If the date is older than 30 days, return the short date "Mar 15".

**2. Create `resources/js/components/admin/admin-virtual-table/admin-virtual-table.vue`:**

This is a presentational component — NO direct store access. All data via props, events via emits.

Props:
- `data: UnifiedMember[]` — the filtered member array
- `columns: ColumnDef<UnifiedMember>[]` — column definitions
- `isLoading: boolean` — show skeleton rows when true

Emits:
- `(e: 'row-click', userId: string): void`

Script setup:
- `const tableContainerRef = ref<HTMLDivElement | null>(null)`
- Create table with `useVueTable({ get data() { return props.data }, get columns() { return props.columns }, getCoreRowModel: getCoreRowModel(), manualFiltering: true })` — CRITICAL: use getter syntax for `data` and `columns` to maintain reactivity.
- Create virtualizer with `useVirtualizer(computed(() => ({ count: table.getRowModel().rows.length, estimateSize: () => 56, getScrollElement: () => tableContainerRef.value, overscan: 5 })))` — wrap options in `computed`.
- `const virtualRows = computed(() => rowVirtualizer.value.getVirtualItems())` — MUST be computed, not direct call.
- `const totalSize = computed(() => rowVirtualizer.value.getTotalSize())`
- `const paddingTop = computed(() => virtualRows.value.length > 0 ? virtualRows.value[0].start : 0)`
- `const paddingBottom = computed(() => virtualRows.value.length > 0 ? totalSize.value - virtualRows.value[virtualRows.value.length - 1].end : 0)`

Template:
- Outer `div.AdminVirtualTable` wrapping `div.AdminVirtualTable__scroll-container` (ref=tableContainerRef)
- `table.AdminVirtualTable__table` with `thead.AdminVirtualTable__thead` rendering header groups from `table.getHeaderGroups()`
- Use `FlexRender` for header cells
- `tbody.AdminVirtualTable__tbody` with spacer-row pattern:
  - Top spacer `<tr>` with single `<td :colspan>` when `paddingTop > 0`
  - `v-for="virtualRow in virtualRows"` rendering visible rows: `<tr class="AdminVirtualTable__row" @click="emit('row-click', table.getRowModel().rows[virtualRow.index].original.userId)">`
  - Each row iterates visible cells with `FlexRender`
  - Bottom spacer `<tr>` with single `<td :colspan>` when `paddingBottom > 0`
- When `isLoading` is true, show 8 skeleton rows instead of the real tbody. Each skeleton row: `<tr class="AdminVirtualTable__skeleton-row">` with 4 `<td>` cells containing `<div class="AdminVirtualTable__skeleton-cell">` of varying widths (32px circle for avatar, 120px for name, 200px for groups, 80px for date).
- Do NOT use `position: absolute` or `transform: translateY()` on `<tr>` elements — this causes border rendering bugs in Chrome/Firefox per research.
