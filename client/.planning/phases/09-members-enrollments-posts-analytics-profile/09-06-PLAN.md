**AdminController.php** — Add memberId to islandProps:
In the `__invoke` method, add `'memberId' => $member['id'] ?? null` to the `$islandProps` array. Place it after `'logoutUrl'`.

**admin-island.vue** — Add memberId prop:
Add `memberId?: string` to the Props interface. No default needed (optional). This prop is now available to any child component via `inject` or direct prop drilling. Since profile-section is a RouterView child and cannot receive props from the parent router, use `provide`:
- In script setup, after props definition: `import { provide } from 'vue'` and `provide('memberId', props.memberId)`
- Also provide memberName and avatarUrl so profile section can read them: `provide('memberName', props.memberName)`, `provide('avatarUrl', props.avatarUrl)`

**profile.ui.ts** — Pinia composition store (ID: `'profile-ui'`):
- Import `inject` (cannot use inject in store — must receive memberId as parameter)
- Actually, Pinia stores cannot use `inject`. Instead, store the memberId in a ref that gets set from the section component.
- Refs: `memberId: ref<string>('')`, `firstName: ref('')`, `lastName: ref('')`, `avatarUrl: ref<string|null>(null)`, `isSaving: ref(false)`, `isUploading: ref(false)`, `error: ref<string|null>(null)`, `successMessage: ref<string|null>(null)`
- Methods:
  - `init(id: string, name: string, avatar?: string)`: parse name into firstName/lastName (split on first space), set memberId, avatarUrl. This is called once from profile-section.vue onMounted.
  - `async saveProfile()`: PATCH `/admin/api/members/${memberId.value}` with `{ firstName: firstName.value, lastName: lastName.value }`. On success set successMessage "Profile updated", clear after 3 seconds. On error set error message.
  - `async uploadAvatar(file: File)`: Create FormData with `avatar` field (key is `'avatar'`). POST `/admin/api/members/${memberId.value}/avatar` with formData. Extract URL from `res.data.data.url`. Update avatarUrl.
- Return all.

**profile-section.vue** — Rewrite from stub:
```vue
<script setup lang="ts">
import { inject, onMounted } from 'vue'
import { useProfileUI } from '../stores/ui/profile.ui'
import AdminImageUpload from '../../../components/admin/admin-image-upload/admin-image-upload.vue'

const memberId = inject<string>('memberId', '')
const memberName = inject<string>('memberName', '')
const memberAvatar = inject<string | undefined>('avatarUrl', undefined)

const ui = useProfileUI()

onMounted(() => {
  ui.init(memberId, memberName, memberAvatar)
})
</script>

<template>
  <div class="AdminSection">
    <header class="AdminSection__header">
      <h1 class="AdminSection__title">Profile</h1>
    </header>
    <div class="AdminSection__body">
      <div class="ProfileSection">
        <!-- Avatar Upload -->
        <AdminImageUpload
          :current-url="ui.avatarUrl ?? undefined"
          :uploading="ui.isUploading"
          label="Profile Picture"
          @upload="ui.uploadAvatar"
        />

        <!-- Name Form -->
        <div class="ProfileSection__form" style="margin-top: 24px">
          <div class="ProfileSection__form-row">
            <label class="ProfileSection__form-label">First Name</label>
            <input v-model="ui.firstName" class="ProfileSection__form-input" placeholder="First name" />
          </div>
          <div class="ProfileSection__form-row">
            <label class="ProfileSection__form-label">Last Name</label>
            <input v-model="ui.lastName" class="ProfileSection__form-input" placeholder="Last name" />
          </div>

          <div v-if="ui.error" class="ProfileSection__error">{{ ui.error }}</div>
          <div v-if="ui.successMessage" class="ProfileSection__success">{{ ui.successMessage }}</div>

          <button class="ProfileSection__save-btn" :disabled="ui.isSaving" @click="ui.saveProfile()">
            {{ ui.isSaving ? 'Saving...' : 'Save Changes' }}
          </button>
        </div>
      </div>
    </div>
  </div>
</template>
```
