**analytics.domain.ts** — Pinia composition store (ID: `'analytics-domain'`):
- Types (export):
  - `HeatmapBucket { day: number, hour: number, count: number }` (day 0-6, hour 0-23)
  - `DayActivityCount { date: string, count: number }`
  - `CalendarEvent { date: string, title: string, groupName: string, enrollmentId: string }`
- State: `heatmapData: ref<HeatmapBucket[]>([])`, `weeklyData: ref<DayActivityCount[]>([])`, `calendarEvents: ref<CalendarEvent[]>([])`, `isLoading: ref(false)`, `isCalendarLoading: ref(false)`, `error: ref<string|null>(null)`, `activeEnrollmentCount: ref(0)`
- Methods:
  - `loadHeatmap()`: GET `/admin/api/activity-logs/stats/heatmap`; set heatmapData from res.data.data
  - `loadWeeklyStats()`: GET `/admin/api/activity-logs/stats`; set weeklyData from res.data.data
  - `loadCalendarEvents()`: This is the N+1 cascade from research. Import `useGroupsDomain` and use axios directly:
    1. Load groups: `const groupsDomain = useGroupsDomain(); await groupsDomain.loadGroups()`
    2. Load enrollments for all groups in parallel: `const enrollmentResults = await Promise.all(groupsDomain.groups.map(g => axios.get(\`/admin/api/groups/${g.id}/enrollments\`).catch(() => ({ data: { enrollments: [] } }))))`
    3. Collect all enrollment IDs: `const allEnrollments = enrollmentResults.flatMap(r => r.data.enrollments ?? [])`
    4. **Cap at 50 enrollments** to prevent unbounded N+1 cascade: `const cappedEnrollments = allEnrollments.slice(0, 50)`
    5. Count active enrollments (where `new Date(e.endDate) >= new Date()`): set `activeEnrollmentCount.value`
    6. Load details for capped enrollments in parallel: `const details = await Promise.all(cappedEnrollments.map(e => axios.get(\`/admin/api/enrollments/${e.id}\`).catch(() => ({ data: { enrollment: { lessonSchedules: [] } } }))))`
    7. Collect all upcoming scheduled lessons: filter where `new Date(schedule.scheduledDate) >= new Date()`. Map to CalendarEvent with groupName from groups lookup.
    8. Sort by scheduledDate ascending. Set calendarEvents.
    NOTE: Use `isCalendarLoading` (separate flag) since this is slow — the rest of the dashboard should render immediately.
  - `loadAll()`: call loadHeatmap + loadWeeklyStats in parallel. Then call loadCalendarEvents (can be async, non-blocking for initial render).
- Return all.

  - `heatmapSeries`: transform heatmapData to ApexCharts series format:
    ```typescript
    const days = ['Sun', 'Mon', 'Tue', 'Wed', 'Thu', 'Fri', 'Sat']
    return days.map((dayName, dayIdx) => ({
      name: dayName,
      data: Array.from({ length: 24 }, (_, hour) => {
        const bucket = domain.heatmapData.find(b => b.day === dayIdx && b.hour === hour)
        return { x: `${hour}:00`, y: bucket?.count ?? 0 }
      })
    }))
    ```
  - `heatmapOptions`: ApexCharts options for type: 'heatmap' — dark theme, transparent background, no toolbar, color range from rgba(124,58,237,0.1) to #7c3aed, no data labels

  - `upcomingEvents`: first 20 from domain.calendarEvents, grouped by date for display
  - `isLoading`: domain.isLoading
  - `isCalendarLoading`: domain.isCalendarLoading
- Methods:
  - `loadDashboard()`: call `groupsDomain.loadGroups()` then `domain.loadAll()`

            <div class="Dashboard__kpi-value">{{ ui.totalMembers }}</div>

            <div class="Dashboard__kpi-label">Active Enrollments</div>

          <div class="Dashboard__chart-card">

          <div class="Dashboard__chart-card">
            <div class="Dashboard__chart-title">Engagement by Day/Hour</div>

              <div class="Dashboard__calendar-group">{{ event.groupName }}</div>

      </div>
    </div>
  </div>
