**enrollments.domain.ts** — Pinia composition store (ID: `'enrollments-domain'`):
- Interfaces (export all):
  - `Enrollment { id, groupId, studyProgramId, startDate, endDate, enabledDays: string, smsTime?, timezone?, requireResponse?, createdAt, updatedAt }`
  - `EnrollmentWithProgram extends Enrollment { studyProgram?: { id, name, coverImageUrl? } }`
  - `LessonSchedule { id, enrollmentId, scheduledDate, title?, lesson?: { dayNumber, title } }`
  - `EnrollmentDetails extends Enrollment { studyProgram?: { id, name, coverImageUrl? }, lessonSchedules: LessonSchedule[] }`
  - `UnenrollInfo { totalLessons, lessonsWithData, cleanLessons }`
  - `CreateEnrollmentPayload { groupId, studyProgramId, startDate, enabledDays: string[], smsTime?, timezone?, requireResponse? }`
- State: `enrollmentsByGroup: ref<Record<string, EnrollmentWithProgram[]>>({})`, `enrollmentDetails: ref<Record<string, EnrollmentDetails>>({})`, `isLoading: ref(false)`, `error: ref<string|null>(null)`, `loadedGroupIds: ref(new Set<string>())`
- Methods:
  - `loadEnrollments(groupId, force = false)`: skip if cached and !force; GET `/admin/api/groups/${groupId}/enrollments`; set enrollmentsByGroup[groupId] from res.data.enrollments; add to loadedGroupIds
  - `loadEnrollmentDetail(enrollmentId)`: GET `/admin/api/enrollments/${enrollmentId}`; set enrollmentDetails[enrollmentId] from res.data.enrollment
  - `createEnrollment(payload: CreateEnrollmentPayload)`: POST `/admin/api/enrollments` with payload; push to enrollmentsByGroup[payload.groupId]; invalidate cache `loadedGroupIds.delete(payload.groupId)`
  - `deleteEnrollment(enrollmentId, groupId)`: DELETE `/admin/api/enrollments/${enrollmentId}`; filter from enrollmentsByGroup[groupId]; delete enrollmentDetails[enrollmentId]
  - `cancelFuture(enrollmentId, groupId)`: POST `/admin/api/enrollments/${enrollmentId}/cancel-future` body `{}`; reload enrollment detail
  - `getUnenrollInfo(enrollmentId): Promise<UnenrollInfo>`: GET `/admin/api/enrollments/${enrollmentId}/unenroll-info`; return res.data.data
  - Return all state and methods

**enrollments-tab.ui.ts** — Pinia composition store (ID: `'enrollments-tab-ui'`):
- Import `useEnrollmentsDomain`, `useProgramsDomain`, `useRoute`
- Refs: `isCreateFormOpen: ref(false)`, `expandedEnrollmentId: ref<string|null>(null)`, `confirmDeleteId: ref<string|null>(null)`, `unenrollInfo: ref<UnenrollInfo|null>(null)`, `formError: ref<string|null>(null)`, `isSaving: ref(false)`
- Also add form refs directly in the store (keeps component thin):
  - `selectedDays: ref<string[]>(['MON','TUE','WED','THU','FRI'])`
  - `createForm: ref({ studyProgramId: '', startDate: '', smsTime: '', timezone: '' })`
- Computeds:
  - `groupId`: `route.params.id as string`
  - `enrollments`: `domain.enrollmentsByGroup[groupId] ?? []`
  - `enrollmentDetail`: if expandedEnrollmentId, `domain.enrollmentDetails[expandedEnrollmentId] ?? null`
  - `programOptions`: `programsDomain.programs.filter(p => p.isPublished).map(p => ({ value: p.id, label: p.name }))`
  - `isLoading`: `domain.isLoading`
- Methods:
  - `loadData()`: call `domain.loadEnrollments(groupId)` + `programsDomain.loadPrograms()` in parallel
  - `openCreate()`: set isCreateFormOpen true, formError null, reset createForm and selectedDays to defaults
  - `closeCreate()`: set isCreateFormOpen false
  - `expandEnrollment(id)`: set expandedEnrollmentId, call domain.loadEnrollmentDetail(id)
  - `collapseEnrollment()`: set expandedEnrollmentId null
  - `requestDelete(id)`: load unenroll info first (`domain.getUnenrollInfo(id)`), set confirmDeleteId, set unenrollInfo
  - `cancelDelete()`: reset confirmDeleteId, unenrollInfo
  - `confirmDelete()`: call domain.deleteEnrollment(confirmDeleteId, groupId), reset confirmDeleteId
  - `handleCancelFuture(id)`: call domain.cancelFuture(id, groupId)
  - `async submitCreate()`: set isSaving true, formError null; build payload from createForm + selectedDays, convert startDate to ISO: `new Date(createForm.value.startDate + 'T00:00:00.000Z').toISOString()`; call `domain.createEnrollment(payload)`; on success call closeCreate(); on error set formError from err.response.data.message or fallback; finally set isSaving false.
- `createFields` computed: returns form field definitions:
  - `studyProgramId` (select, required, options from programOptions)
  - `startDate` (text, required, placeholder "YYYY-MM-DD")
  - `smsTime` (text, placeholder "HH:MM (e.g. 08:00)")
  - `timezone` (select, options for common US timezones: America/New_York, America/Chicago, America/Denver, America/Los_Angeles, America/Phoenix, America/Anchorage, Pacific/Honolulu)
  NOTE: `enabledDays` does NOT fit AdminForm — will be handled inline in the section template with custom checkboxes.
