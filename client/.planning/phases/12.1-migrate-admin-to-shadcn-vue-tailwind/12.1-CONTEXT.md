# Phase 12.1: Migrate Admin to shadcn-vue + Tailwind - Context

**Gathered:** 2026-03-21
**Status:** Ready for planning

<domain>
## Phase Boundary

Install Tailwind CSS (scoped to admin island only), initialize shadcn-vue, and replace ALL existing BEM admin components with shadcn-vue equivalents. Every admin section view is updated to use the new components. Pinia stores are untouched — only the template/style layer changes. Member-facing Blade pages remain SCSS/BEM. After this phase, the admin panel looks and feels like a modern shadcn app with dark theme.

</domain>

<decisions>
## Implementation Decisions

### Tailwind Scoping
- Tailwind CSS configured for the admin island only
- Use Tailwind's `content` config to scan only `resources/js/islands/admin-island/**` and `resources/js/components/admin/**` and the shadcn component directory
- Blade/SSR pages (`resources/views/**`) are NOT scanned — existing SCSS/BEM is preserved
- Tailwind loaded via a separate `admin.css` entry point (not mixed into `app.scss`)
- `vite.config.js` updated to include `admin.css` as an additional entry
- Admin Blade layout (`resources/views/layouts/admin.blade.php`) loads `admin.css` via `@vite`

### shadcn-vue Setup
- Initialize via `npx shadcn-vue@latest init` — this creates `components.json` and a `ui/` directory
- shadcn components go in `resources/js/components/ui/` (shared between admin components)
- Use shadcn's default dark theme (zinc palette) — it's a good fit for the existing dark admin aesthetic
- CSS variables approach (not Tailwind classes) for theme — enables future theming

### Component Migration Map
| Current BEM | shadcn-vue Replacement | Notes |
|---|---|---|
| `AdminTable` | `<Table>` components | shadcn Table + custom row click handling |
| `AdminVirtualTable` | Keep TanStack Virtual + wrap in shadcn `<Table>` styles | Hybrid — TanStack for virtualization, shadcn for visual |
| `AdminForm` | shadcn `<Form>`, `<Field>`, `<Input>`, `<Select>`, `<Button>` | vee-validate integration if shadcn-vue uses it |
| `AdminConfirmDialog` | shadcn `<AlertDialog>` | Direct replacement |
| `AdminImageUpload` | shadcn `<Input type="file">` + `<Avatar>` for preview | Custom wrapper around shadcn primitives |
| `AdminSidebar` | shadcn `<Sidebar>` (sidebar-07 pattern — collapses to icons) | Major upgrade — built-in collapse, responsive, mobile sheet |
| `AdminActivityList` | shadcn `<Accordion>` + `<Badge>` | Activity type badges + collapsible panels |
| `MemberFilterBar` | shadcn `<Combobox>` + `<Badge>` | Replace reka-ui direct usage with shadcn wrapper |
| `MemberProfileDrawer` | shadcn `<Sheet>` (side="right") | Slide-over panel with proper animation |
| Admin tabs | shadcn `<Tabs>` | Replace BEM tab styling |
| Loading states | shadcn `<Skeleton>` | Replace shimmer animations |
| Buttons everywhere | shadcn `<Button>` variants | Primary, secondary, destructive, ghost, outline |
| Section page titles | shadcn typography classes | `text-2xl font-bold tracking-tight` |

### Section View Updates
- All 5 section views (Dashboard, Groups, Programs, Profile, Members) are updated to use shadcn components
- Template markup changes from BEM divs to shadcn component imports
- Pinia store interfaces (props/emits contracts) remain identical
- Section-level SCSS files are deleted — replaced by Tailwind utility classes

### SCSS Cleanup
- All 16 admin SCSS files in `resources/css/components/admin/` are deleted
- `app.scss` `@use` imports for admin components are removed
- Admin layout SCSS (`admin-layout.scss`, `admin-sidebar.scss`) replaced by shadcn Sidebar + Tailwind

### What Stays
- All Pinia stores (domain + UI) — untouched
- TanStack Table + Virtual integration — kept, restyled with shadcn Table classes
- reka-ui — still a dependency (shadcn-vue uses it internally), but no more direct imports in admin components
- Vue Router configuration — untouched
- Admin Blade layout structure — updated to load `admin.css` alongside `app.css`

### Claude's Discretion
- Exact shadcn component variants used (e.g., Button variant="destructive" vs variant="outline")
- Whether to use shadcn's built-in form validation (vee-validate) or keep current pattern
- Sidebar collapse breakpoint and mobile behavior
- Exact Tailwind content paths in tailwind.config
- Whether admin.css is a separate Vite entry or loaded via a Tailwind @layer import
- Order of component migration (could do sidebar first, or tables first, etc.)

</decisions>

<specifics>
## Specific Ideas

- The sidebar-07 shadcn block (collapses to icons) is the closest match to the current admin sidebar behavior — use it as the starting point
- shadcn Sheet component (side="right") is a perfect replacement for MemberProfileDrawer — includes built-in animation, backdrop, close on escape
- The dark theme should feel like a native shadcn app, not a custom theme bolted on — use shadcn's zinc dark defaults
- ApexCharts (dashboard analytics) can stay as-is — just needs the container to use Tailwind classes instead of BEM

</specifics>

## Existing Code Insights

- `resources/js/components/ui/` — shadcn component output directory
