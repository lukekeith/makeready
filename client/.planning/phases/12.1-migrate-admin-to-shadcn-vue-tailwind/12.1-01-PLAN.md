  <action>
1. Install Tailwind v4 and vee-validate dependencies:
   ```bash
   cd /Users/lukekeith/www/makeready/client
   npm install tailwindcss @tailwindcss/vite
   npm install vee-validate @vee-validate/zod
   npm install lucide-vue-next
   ```

2. Create `resources/css/admin.css` with scoped Tailwind entry:
   ```css
   @import "tailwindcss" source(none);
   @source "../js/islands/admin-island/**/*.vue";
   @source "../js/components/admin/**/*.vue";
   @source "../js/components/ui/**/*.vue";
   ```
   Per user decision: Tailwind scoped to admin island only via source(none). Blade/SSR pages NOT scanned.

3. Update `vite.config.js`:
   - Add `import tailwindcss from '@tailwindcss/vite'` at top
   - Add `'resources/css/admin.css'` to the laravel input array (after app.scss, before app.js)
   - Add `tailwindcss()` plugin after `vue()` plugin
   - Keep all existing config (aliases, scss options, server watch) unchanged

4. Run `npx shadcn-vue@latest init` — when prompted, configure:
   - Style: new-york
   - Base color: zinc
   - CSS variables: yes
   - CSS file: resources/css/admin.css
   - Tailwind config: (leave empty — Tailwind v4 has no config file)
   - Components alias: @/components
   - Utils alias: @/lib/utils
   - Composables alias: @/composables
   - UI alias: @/components/ui

   If the CLI is interactive and cannot be automated, manually create `components.json` at project root with the configuration from RESEARCH.md Pattern 2, and create `resources/js/lib/utils.ts` with the `cn()` utility function:
   ```typescript
   import { type ClassValue, clsx } from 'clsx'
   import { twMerge } from 'tailwind-merge'
   export function cn(...inputs: ClassValue[]) { return twMerge(clsx(inputs)) }
   ```
   Install clsx and tailwind-merge if not already present: `npm install clsx tailwind-merge`

5. Install all required shadcn components in one pass:
   ```bash
   npx shadcn-vue@latest add sidebar sheet alert-dialog table combobox accordion badge skeleton button input avatar tabs form select textarea label separator dropdown-menu tooltip scroll-area command popover dialog
   ```
   If the CLI is interactive, run with `--yes` or `--overwrite` flag. If that fails, add components one at a time.
