  <files>app/Http/Controllers/AdminApiProxyController.php, routes/web.php</files>
  <action>
Create `app/Http/Controllers/AdminApiProxyController.php` with:
- Constructor injects `ApiService $api`
- `handle(Request $request, string $path)` method that:
  - Builds endpoint as `/api/{$path}` with query string appended if present
  - Uses `match(strtolower($request->method()))` to dispatch to correct ApiService method:
    - `'get'` -> `$this->api->get($endpoint, $request)`
    - `'post'` -> `$this->handlePost($request, $endpoint)` (separate method for multipart detection)
    - `'patch'` -> `$this->api->patch($endpoint, $request->json()->all(), $request)`
    - `'put'` -> `$this->api->patch($endpoint, $request->json()->all(), $request)` (treat PUT same as PATCH)
    - `'delete'` -> `$this->api->delete($endpoint, $request)`
    - default -> return 405
  - Returns `response()->json($result['body'], $result['status'])` with Set-Cookie headers forwarded
- Private `handlePost(Request $request, string $endpoint)` method:
  - If `$request->hasFile('image')`: call `$this->api->upload($endpoint, 'image', $request->file('image'), $request)`
  - Otherwise: call `$this->api->post($endpoint, $request->json()->all() ?? [], $request)`

In `routes/web.php`:
- Add `use App\Http\Controllers\AdminApiProxyController;` import at top
- Inside the admin route group, add BEFORE the catch-all `/{any?}` line:
  ```php
  Route::match(['GET', 'POST', 'PATCH', 'PUT', 'DELETE'], '/api/{path}', [AdminApiProxyController::class, 'handle'])
      ->where('path', '.*')
      ->name('api.proxy');
  ```
- Remove the comment "Phase 6+ will add API proxy routes here" and replace with "API proxy — must come before the catch-all Blade route"

CRITICAL: The proxy route MUST be registered before the `/{any?}` catch-all. If reversed, all GET requests to /admin/api/* will match the catch-all and return HTML.
