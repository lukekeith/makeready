**groups.domain.ts** — follows composition API `defineStore` pattern from modal.store.ts:

- Export `Group` interface: `{ id: string, name: string, description?: string, coverImageUrl?: string, isPrivate: boolean, allowInvites: boolean, memberDirectory: boolean, welcomeMessage?: string, ageRange?: { min?: number, max?: number }, maxMembers?: number, memberCount?: number }`
- Export `CreateGroupPayload`: `{ name: string, description?: string, isPrivate?: boolean, allowInvites?: boolean, memberDirectory?: boolean }`
- Export `UpdateGroupPayload`: all Group fields except `id` and `memberCount`, all optional
- Store ID: `'groups-domain'`
- Refs: `groups: ref<Group[]>([])`, `isLoading: ref(false)`, `error: ref<string | null>(null)`
- Methods:
  - `loadGroups()`: GET `/admin/api/groups`, set `groups.value = res.data.groups ?? []`
  - `getGroup(id)`: GET `/admin/api/groups/${id}`, upsert into `groups.value` array
  - `createGroup(payload)`: POST `/admin/api/groups`, push returned group to `groups.value`
  - `updateGroup(id, payload)`: PATCH `/admin/api/groups/${id}`, splice-replace in `groups.value`
  - `deleteGroup(id)`: DELETE `/admin/api/groups/${id}`, filter out from `groups.value`
  - `uploadCoverImage(id, file: File)`: POST with FormData (NO explicit Content-Type header — let axios set boundary), upsert coverImageUrl
- All methods wrap in try/catch, set `error.value` on failure
- Return all refs and methods

**groups-list.ui.ts** — consumes domain store:

- Store ID: `'groups-list-ui'`
- Import `useGroupsDomain` and `useRouter` from vue-router
- Refs: `isCreateFormOpen: ref(false)`, `editingGroupId: ref<string | null>(null)`, `confirmDeleteId: ref<string | null>(null)`, `formError: ref<string | null>(null)`
- Computeds:
  - `tableColumns`: returns `['Name', 'Members', 'Privacy']`
  - `tableRows`: maps domain.groups to `{ id, cells: [name, memberCount display, privacy], coverImageUrl, badge }`
  - `editingGroup`: finds group by editingGroupId
  - `confirmDeleteGroup`: finds group by confirmDeleteId
  - `isEditing`: `editingGroupId.value !== null`
- Methods:
  - `openCreateForm()`, `openEditForm(id)`, `closeForm()`, `requestDelete(id)`, `cancelDelete()`, `navigateToDetail(id)` (uses router.push)
- Return all refs, computeds, and methods

Rewrite the existing stub `groups-section.vue` to implement the full groups list view.

This component is the ONLY component that imports Pinia stores. It orchestrates domain store calls and passes data to pure display components via props.

Script setup:
- Import `onMounted, ref` from vue, `useRoute` from vue-router
- Import `useGroupsDomain` from stores/domain/groups.domain
- Import `useGroupsListUI` from stores/ui/groups-list.ui
- Import AdminTable, AdminForm, AdminConfirmDialog from `../../../components/admin/`
- Initialize both stores
- `const route = useRoute()`
- `const isSaving = ref(false)`

- `onMounted`: if no `route.params.id`, call `domain.loadGroups()`

- Define `createFields` and `editFields` arrays for AdminForm:
  - Create: `name` (text, required), `description` (textarea)
  - Edit: same fields

- `handleCreate(payload)`: set isSaving, call `domain.createGroup(payload)`, call `listUI.closeForm()`, catch sets `listUI.formError`
- `handleUpdate(payload)`: set isSaving, call `domain.updateGroup(listUI.editingGroup.id, payload)`, call `listUI.closeForm()`, catch sets `listUI.formError`
- `handleDelete()`: call `domain.deleteGroup(listUI.confirmDeleteGroup.id)`, always call `listUI.cancelDelete()` in finally block

Template (list view — when `!route.params.id`):
```html
<div class="AdminSection">
  <header class="AdminSection__header">
    <h1 class="AdminSection__title">Groups</h1>
    <button class="AdminSection__action-btn" @click="listUI.openCreateForm">
      + Create Group
    </button>
  </header>
  <div class="AdminSection__body">
    <AdminTable
      :columns="listUI.tableColumns"
      :rows="listUI.tableRows"
      :loading="domain.isLoading"
      empty-message="No groups yet. Create your first group."
      @row-click="listUI.navigateToDetail"
      @edit="listUI.openEditForm"
      @delete="listUI.requestDelete"
    />
  </div>

  <!-- Create/Edit Form Modal -->
  <AdminForm
    :key="listUI.editingGroupId ?? 'create'"
    :open="listUI.isCreateFormOpen"
    :title="listUI.isEditing ? 'Edit Group' : 'Create Group'"
    :fields="listUI.isEditing ? editFields : createFields"
    :values="listUI.isEditing ? { name: listUI.editingGroup?.name, description: listUI.editingGroup?.description ?? '' } : {}"
    :error="listUI.formError"
    :saving="isSaving"
    @save="listUI.isEditing ? handleUpdate($event) : handleCreate($event)"
    @cancel="listUI.closeForm"
  />

  <!-- Delete Confirmation -->
  <AdminConfirmDialog
    :open="!!listUI.confirmDeleteGroup"
    title="Delete Group"
    :message="`Are you sure you want to delete &quot;${listUI.confirmDeleteGroup?.name ?? ''}&quot;? This cannot be undone.`"
    @confirm="handleDelete"
    @cancel="listUI.cancelDelete"
  />
</div>
```
