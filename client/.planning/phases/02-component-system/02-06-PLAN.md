Purpose: This is the core of the architecture change — converting presentation-only components from client-side Vue to server-rendered Blade. The result is identical visual output but rendered by PHP on the server, making content visible to crawlers without JavaScript.

**For EVERY component, follow this pattern:**
```php
@props([
    'propName' => 'defaultValue',
    // ... all props from Vue's defineProps/withDefaults
])

@php
    // Only if component has CVA variants:
    $classes = cva('BlockName', [
        'variants' => [
            'variantKey' => [
                'Value' => 'BlockName--value',
            ],
        ],
    ], ['variantKey' => $variantKey]);
@endphp

<element {{ $attributes->merge(['class' => $classes]) }}>
    {{-- Template content translated from Vue --}}
    {{ $slot }}
</element>
```

**Key translation rules:**
- Vue `v-if` -> Blade `@if`
- Vue `v-for="item in items"` -> Blade `@foreach($items as $item)`
- Vue `<slot />` -> Blade `{{ $slot }}`
- Vue `<slot name="foo" />` -> Blade `{{ $foo ?? '' }}` (check with `@isset($foo)`)
- Vue `:class="classnames('Foo', props.class)"` -> Blade `$attributes->merge(['class' => 'Foo'])`
- Vue `@click="handler"` -> REMOVE (no JS events in Blade; callers wrap in `<a>` or `<form>`)
- Vue `v-bind="$attrs"` -> Blade `{{ $attributes }}` or `$attributes->merge([...])`
- Vue `defineEmits` -> REMOVE entirely (no events in Blade)
- Any `import` statements -> REMOVE (Blade has no imports)

**IMPORTANT:** Do NOT create Blade versions of the 4 interactive primitives (digit, modal, verify-code, bullet-text-input) — those stay as Vue SFCs.
