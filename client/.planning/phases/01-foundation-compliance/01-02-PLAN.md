    - Test 4: ApiService::post() forwards Cookie header from incoming request
    - Test 5: Return array includes 'status', 'body', and 'setCookies' keys
    - Test 6: Single Set-Cookie header from API response is captured in return value
    - Test 7: Multiple Set-Cookie headers from API response are ALL captured (not just the first)
    - Test 8 (Feature): A controller using ApiService passes all Set-Cookie headers back to browser response

  <action>
    **Step 1: Write tests first**

    Create `tests/Unit/ApiServiceTest.php` using Pest:
    - Use `Http::fake()` to mock the external API
    - Verify correct URL construction from config
    - Verify Cookie header forwarding
    - Verify Set-Cookie capture in return value
    - Verify both GET and POST methods
    - CRITICAL: Include a test where the API response returns multiple Set-Cookie headers (e.g., session cookie + CSRF cookie). Assert that setCookies is an array containing ALL of them, not just the first.

    Create `tests/Feature/CookieProxyTest.php`:
    - Create a test route that uses ApiService and returns a response
    - Use `Http::fake()` with a response that includes multiple Set-Cookie headers
    - Verify the Laravel response includes ALL Set-Cookie headers from the API

    **Step 2: Implement ApiService**

    Create `app/Services/ApiService.php` per RESEARCH.md Pattern 5:
    - Constructor reads `config('services.makeready.url')`
    - `get(string $endpoint, Request $request): array` — forwards Cookie header, returns status/body/setCookies
    - `post(string $endpoint, array $data, Request $request): array` — same with POST body

    CRITICAL: Handle multiple Set-Cookie headers. The Http facade's `->header('Set-Cookie')` returns only the first. Use `->headers()` to get the full headers collection, then extract all Set-Cookie values. The `setCookies` return value MUST be an array of strings (one per Set-Cookie header).

    CRITICAL: Per Pitfall 6 in RESEARCH.md — do NOT forward the entire `$request->header('Cookie')` string blindly. Extract only the relevant API session cookies by name. For now, forward the full Cookie header (since this is a fresh Laravel app with no encrypted cookies yet), but add a TODO comment noting this should be refined once the actual API cookie name is confirmed.

    **Step 3: Verify tests pass**

    Run `php artisan test --filter "ApiServiceTest|CookieProxyTest"`

    Create `app/Http/Middleware/CheckMemberSession.php` per RESEARCH.md Pattern 6:

    - Call `$this->api->get('/api/members/session', $request)`
