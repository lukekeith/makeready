# Phase 3.1: Visual Fidelity - Context

**Gathered:** 2026-03-18
**Status:** Ready for planning

<domain>
## Phase Boundary

Rewrite all Blade components and page templates to produce HTML and CSS output identical to the original React app. Every page must look the same — same layout, same component structure, same spacing, same colors, same buttons, same visual hierarchy. This is a gap-closure phase fixing the visual mismatch discovered during Phase 4 testing.

</domain>

<decisions>
## Implementation Decisions

### Fidelity Standard
- Every page must be visually identical to the React app output
- Same HTML structure (same DOM nesting, same BEM class names, same semantic elements)
- Same CSS (SCSS files from archive are the source of truth)
- Same component composition (if React uses `<GroupCard>` inside a flex container, Blade must too)
- Same responsive behavior at all breakpoints

### Reference Source
- `archive/react-spa` branch has all original React source code
- Read each React component's TSX to understand exact HTML output and props
- Read each React page's TSX to understand exact component composition and layout
- Read each SCSS file for exact styles
- The existing Vue component versions on main (from Phase 2 Wave 1-2, before cleanup) can also serve as translation reference since they were 1:1 ports

### Scope
- ALL 52 Blade components must produce identical HTML to their React equivalents
- ALL page templates must compose components identically to React pages
- ALL page-level SCSS (from `archive/react-spa:src/pages/*/`) must be migrated
- Fix any missing SCSS imports in app.scss
- Fix any component props that don't match React interfaces

### Process
- For each component: read React TSX → compare to current Blade → rewrite Blade to match
- For each page: read React page TSX → compare to current Blade page → rewrite to match exact composition
- Copy any missing page-level SCSS from archive
- Test by running dev server and visually comparing

### Claude's Discretion
- Order of component/page fixes
- How to batch the work into plans
- Whether to use a systematic component-by-component approach or page-by-page approach
- Test strategy for verifying visual fidelity

</decisions>
