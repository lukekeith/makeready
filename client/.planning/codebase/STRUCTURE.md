## Directory Purposes

**`.storybook/`:**
- Purpose: Storybook configuration for component development and documentation
- Contains: Main.ts config, preview.ts for decorators, manager.ts for theming
- Key files: `main.ts` configures story discovery and addon setup

**`ui/`:**
- Purpose: Centralized UI component library - all reusable components live here
- Contains: Organized into primitive/domain/layout/panel categories with stories
- Key files: `index.ts` barrel export (used in pages via import from 'ui')
- CRITICAL: Never put business logic or store access in these components

**`util/`:**
- Purpose: Shared utilities available to UI components and pages
- Contains: CVA wrapper, classnames helper, when() conditional, custom hooks
- Key files: `index.ts` barrel export, `cva.ts` custom wrapper, `hooks/` directory

**`src/pages/`:**
- Purpose: Page components for each route - these connect stores to UI
- Contains: One folder per page with [name].page.tsx and [name].scss
- Pattern: Each page uses useLifecycle(Application.ui.[storeName]) to initialize
- Access: Import UI components from 'ui' barrel, stores from '@/store/ApplicationStore'

**`src/store/`:**
- Purpose: MobX state management organized by concern
- Domain: Raw API data, handles all apiClient calls
- Session: Authentication state, member info
- UI: Component-level state, computed props for UI
- Key files: `ApplicationStore.ts` root singleton, `DomainStore.ts` and `UIStore.ts` containers

**`src/lib/`:**
- Purpose: Application-specific utilities (not shared with other apps)
- Key files: `api-client.ts` singleton for HTTP, includes session cookies automatically

**`src/components/`:**
- Purpose: App-specific components that don't belong in ui/ (e.g., ModalRegistry)
- Contains: Components used only by specific pages, not library components
- Example: `ModalRegistry.tsx` maps modal IDs to content for ModalProvider

## Key File Locations

**Entry Points:**
- `src/main.tsx`: React DOM root, creates Application singleton
- `src/App.tsx`: React Router routes, ModalRegistry provider setup
- `index.html`: HTML template with root div

**Configuration:**
- `vite.config.ts`: Path aliases (@, ui, util, shared), port 5180
- `tsconfig.json`: Path aliases for TypeScript, exclude stories from build
- `tailwind.config.js`: Tailwind design tokens configuration
- `.env`: VITE_API_URL for API backend

**Core Logic:**
- `src/store/ApplicationStore.ts`: Root state container
- `src/lib/api-client.ts`: HTTP client with credentials
- `util/cva.ts`: Custom CVA wrapper for type-safe variants
- `ui/index.ts`: Component barrel export (source of truth for UI APIs)

**Testing:**
- `e2e/`: Playwright E2E tests
- `ui/stories/`: Storybook stories for components
- `src/store/__tests__/`: Store unit tests
- `util/__tests__/`: Utility unit tests

## Naming Conventions

**Files:**
- Pages: `[name].page.tsx` in `src/pages/[name]/`
- Stores: `[name].ts` in `src/store/` or `src/store/[domain|ui]/`
- Domain stores: `[entity].domain.ts` (e.g., `groups.domain.ts`, `auth.domain.ts`)
- UI stores: `[entity|feature].ui.ts` (e.g., `groups-list.ui.ts`, `modal.service.ts`)
- Styles: `[name].scss` in same directory as component
- Stories: `[name].stories.tsx` in `ui/stories/components/[category]/[name]/`

**Directories:**
- Page folders: lowercase with hyphens (e.g., `public-home`, `group-home`, `study-join`)
- Component categories: lowercase plural (primitive, domain, layout, panel)
- Store types: lowercase (domain, ui, session)
- Utilities: lowercase with hyphens (e.g., `use-lifecycle`, `useMobileAppMode`)

**TypeScript:**
- Interfaces: prefix with `I` (IButton, IGroup, IApplicationStore)
- Classes: PascalCase (ApplicationStore, AuthDomain, GroupsListUI)
- Constants: SCREAMING_SNAKE_CASE
- Enums: PascalCase with PascalCase values
- Functions: camelCase

## Where to Add New Code

**New Feature with UI:**
- Primary code: `ui/components/domain/[feature-name]/[feature-name].tsx` (component)
- Styling: `ui/components/domain/[feature-name]/[feature-name].scss` (same folder)
- Story: `ui/stories/components/domain/[feature-name].stories.tsx`
- Export: Add to `ui/index.ts` barrel
- Page: `src/pages/[page-name]/[page-name].page.tsx`
- Store: `src/store/ui/[feature].ui.ts` for page state

**New API Domain:**
- Store: `src/store/domain/[entity].domain.ts` (handles all API calls)
- Reference: Add to `src/store/DomainStore.ts` constructor
- Types: Define `interface I[Entity]` at top of domain file or in `shared/types/`

**New Route/Page:**
- Page file: `src/pages/[route-name]/[route-name].page.tsx`
- Route mapping: Add to `src/App.tsx` Routes config
- Store: `src/store/ui/[feature].ui.ts` if page needs UI state
- Reference: Add to `src/store/UIStore.ts` constructor

**Utilities:**
- Shared across components: `util/` (functions, hooks)
- App-specific: `src/lib/` (API client, app utilities)
- Form validation: `util/validators/` (imported by components)

**Global Styles:**
- Component-level: In component's SCSS file with BEM naming
- Global variables/mixins: `ui/styles/` directory
- Tailwind overrides: `tailwind.config.js`
