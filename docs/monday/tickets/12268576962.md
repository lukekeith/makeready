# calendar breaks when unpublishing

## Problem
After publish → enroll → unpublish → re-enroll, days the leader adds to the study never show up for the member: at the time of the report, a day-count increase created new `Lesson` rows on the live study but nothing back-filled the enrollment snapshots members actually render. "Fixed" looks like: a lesson added after enrollment reaches the enrolled group's calendar (today via the study-sync scheduled-lesson path). Study-sync (PR #1) likely resolved this — needs a device/browser check before closing.

## Reports (verbatim)
- (2026-06-13) "Once the study is published and enrolled, I can unpublish and re-enroll and then add days and they don't show up on the study for the member."

## Root cause
Per CF-5 ([code-fixes.md](../code-fixes.md)), at report time: `PATCH /api/programs/:id` day-increase created new `Lesson` rows (`programs.ts:935-960`) but never back-filled `LessonSchedule`/`ScheduledLessonActivity` for existing enrollments; the only propagation path was a manual per-lesson `POST /enrollments/:id/schedules` (`enrollments.ts:4062`). Study-sync reworked propagation into explicit versioned schedules with opt-in sync.
