# add bible verse unclear

## Problem
On the iPhone app's **Edit Activity modal for a READ activity**, adding a Bible verse is not discoverable. A fresh Read activity opens with an empty custom-text editor already on screen (server-seeds a default unlocked read block on creation) and the add-block affordance is a bare unlabeled `+` icon — nothing visible says a Bible verse can be added until the `+` is tapped. The reporter proposed two alternatives: make the `+` obvious, **or** start the activity without the custom text field up.

**Key deep-dive finding:** the two-option source menu behind the `+` ("Bible verse" / "Custom text") **already existed in the repo at the 2026-05-20 squash — three weeks before this ticket**. So either the reporter's TestFlight build was stale, or (the durable reading) the menu alone doesn't solve it: because an empty editor is already up and the `+` is unlabeled, users start typing in the text block and never discover the menu. "Fixed" observably looks like: a new Read activity where the Bible-verse path is visible without any exploratory tap — e.g. no pre-seeded block + labeled add affordance(s), or a labeled/split add button ("Add Bible verse").

## Reports (verbatim)
- (2026-06-13) "I am unsure every time how to add a bible verse here. The + only does one thing. Let's make it obvious. Or start without the custom text field up"

### Cross-app impact
- **Option A (iPhone UI only — label/split the `+`):** none.
- **Option B (stop seeding the default block, `programs.ts:2322`):** affects BOTH consumers — the web read editor pane will also start rendering zero-block READ activities and must handle the empty state; AND the publish guard `findIncompleteReadActivityDay` (`programs.ts:895-902`) treats content-less READ activities as incomplete, so activities left with zero blocks change publish behavior. Any resolution choosing option B must account for both.

## Root cause
Discoverability, not a defect: `programs.ts:2322-2332` seeds every new READ activity with an empty unlocked read block, which `EditReadActivityPage` renders as an open "Write content..." editor; the only path to a Bible verse is the icon-only `+` (`EditReadActivityPage.swift:449-457`) whose two-card menu (`:512-535`) is invisible until tapped. The pre-opened editor pulls users into typing before they ever try the `+`.
