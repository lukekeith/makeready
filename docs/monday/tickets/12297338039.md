## Root cause
Per [code-fixes.md](../code-fixes.md) and the 2026-07-19 triage:
- **CF-15:** last-verse selection extends to end-of-block text, brushing adjacent-highlight whitespace and tripping the server overlap 400 (`programs.ts:2816-2819`); since guarded by range clamping (`ExegesisVerseView.swift:495-656`).
- **CF-16:** non-idempotent markdown↔attributed round-trip re-emitting drifted content per keystroke (`RichTextInput.swift:70-104`); markdown editor since reworked.
- **CF-14:** server caps titles at 200 chars; no client-side guard — still can't save, only the message improved.
- **UX-6:** exegesis uses native long-press selection while read uses tap-per-verse; interaction redesign needed.
- **CF-8:** unenroll-info had no ownership filter while DELETE required `enrollmentManageFilter` → affordance shown, delete 404s; org-authorization fix `3afaa00` likely resolved (the 6/23–6/27 re-reports were 2–6 days after the fix, likely stale deploy).
- **CF-17:** two-stage modal dismiss with a suspected lingering overlay scrim + missing `state.persist()`; unreconciled with the 2026-07-19 "no transfer flow found" note.
