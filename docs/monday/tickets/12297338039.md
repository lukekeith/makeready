## Root cause
- **Sub-1 (CF-15):** server rejects overlapping exegesis highlights (`programs.ts:3034-3036`). `e8e9718` merges overlapping *notes* but a new *highlight* that overlaps still 400s; whether the last verse's selection over-extends into an adjacent highlight/whitespace needs a device repro.
- **Sub-2:** search-as-you-type sends a request per keystroke and the server records each into recent searches → partial queries pollute history. Fix = save-on-submit / prefix-collapse.
- **Sub-3 (CF-14):** no client-side title length guard; server zod caps 200 → save 400s ("Couldn't save changes").
- **Sub-4 (UX-6):** rich-text editor's selection gesture differs from the native tap-and-hold users expect.
- **Sub-5:** stale enrollment card (dual-store cache gap) → `getUnenrollInfo` 404s. **Fixed `331ca4a`** (ticket 12344966891). NOT an authorization failure — the unenroll-info endpoint has no ownership filter (`enrollments.ts` findUnique by id), so "not found" = genuinely deleted.
- **Sub-6 (CF-17):** transfer completes server-side + updates local cards, but the screen freezes — a modal/overlay dismiss issue post-transfer (and `transferMember` omits `state.persist()`).
