# member - video

## Problem
Four UX defects on the **member web lesson player** (mobile browser, `app.makeready.org`) on a **video/YouTube step**:

1. **The top nav is hidden** — the study title ("test program") is clipped off the top of the screen. The lesson-player header has no `env(safe-area-inset-top)` inset despite the page using `viewport-fit=cover`, so its top edge sits behind the browser status bar / notch.
2. **A "strange message" appears** — the body reads **"No video URL configured"**. This is the `youtube-step.vue` fallback shown to a member when a YOUTUBE activity has no URL. (The reporter's "test program" has a YouTube activity with no URL pasted — a content gap — but surfacing a raw developer-style message to a member is itself the UX defect.)
3. **It's unclear how to get to the next section** — the only forward affordance is the small `>` chevron in the header nav pill; there is no in-content Continue button. (On a YouTube step `canProceed` is `true`, so the chevron is active/white — but its role isn't obvious.)
4. **The UI doesn't scroll** — the player is a fixed `100dvh` app shell with a sticky header and no page scroll. Combined with the clipped header (defect 1), the top of the nav can't be revealed by scrolling.

"Fixed" observably: the header renders clear of the browser chrome (title fully visible), the empty-video state shows a member-appropriate message (or no bare error), and there's a clear forward affordance to advance to the next activity.

## Root cause
- **Hidden nav (1):** `member-lesson-header.scss` opens with flat `padding: 8px` and no top safe-area inset; `layouts/home.blade.php` sets `viewport-fit=cover`, so the header's top edge extends under the status bar/notch and the title is clipped. (`--safe-top` etc. exist in `_tokens.scss:201-204` but are unused by the player.)
- **Strange message (2):** `youtube-step.vue:66-68` renders a bare `"No video URL configured"` div whenever a YOUTUBE activity has neither `youtubeVideoId` nor a parseable `youtubeUrl`. Member-facing, this reads as a broken screen. (The underlying data gap: the "test program" YouTube activity has no URL — authoring, not code.)
- **Unclear next (3):** the sole forward affordance is the header `>` chevron (`member-lesson-header.vue:106-115`); there is no in-content Continue button. On YouTube steps `canProceed` is forced `true` (`youtube-step.vue:31-33`), so the chevron is active but its "advance" role isn't discoverable.
- **No scroll (4):** `.LessonActivity` is a fixed `height: 100dvh` shell (`lesson-island.scss:6-12`) with a sticky header and `overflow:auto` only on `__content`; there is no page-level scroll to reveal the clipped header top.
