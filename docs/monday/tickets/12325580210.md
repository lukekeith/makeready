## Root cause
- **Sub 1 (UX-6):** exegesis uses native long-press/drag selection while read lessons use the lighter tap-per-verse model; no edit path existed for committed highlights. Edit path since added (`exegesisHighlightActionMenu`, `EditExegesisActivityPage.swift:347-382`); selection model unchanged.
- **Sub 2 (CF-1):** the delete guard allows `creatorId == user` **or** `organizationId IN getManageableOrgIds(user)`, and `getManageableOrgIds` (`permission.ts:277-311`) includes any org where the user holds *any* role — so any leader can soft-delete any study in their org. Cross-org deletion is already blocked. Since the org-role authorization shipped (`8efbb7c`, `3afaa00`), this is intentional org-scoping with no role granularity — a policy decision.
