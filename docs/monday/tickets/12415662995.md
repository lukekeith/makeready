# member - reading issue

## Problem
In the member web lesson player's **exegesis step**, the top of the Bible verse and its address (reference) are hidden under the lesson header — the header is "too big and often gets in the way". Cause: the exegesis step hardcodes a 120px top inset while the header publishes its real (larger, dynamic) height to a CSS variable that the read step consumes but exegesis never adopted; it also ignores `safe-area-inset-top`. This is the same defect as 12386101354's second report (CF-10) and part of the 4-ticket safe-area cluster. "Fixed" observably: with the header at any height, the verse's first line and its reference are fully visible.

## Reports (verbatim)

## Root cause
Per CF-10 ([code-fixes.md](../code-fixes.md)) and the 2026-07-19 triage: `exegesis-step.vue:607` hardcodes `padding: 120px 16px 24px` (plus the fade mask at :611-625) while the header is dynamic and publishes its real height to `--member-lesson-header` (`member-lesson-header.vue:10-17`); the static 120px is less than the expanded header height, and `safe-area-inset-top` is ignored. Fix: `calc(var(--member-lesson-header, 200px) + 16px)` + safe-area, mirroring `read-step.vue:113`.
