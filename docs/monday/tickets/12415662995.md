# member - reading issue

## Problem
On the member web lesson player's **exegesis step** (mobile browser), the lesson header overlaps the top of the passage — the reporter "can't see the top of the bible verse … the top nav is too big and it often gets in the way." Two threads:

1. **Header overlaps the top of the verse content (confirmed defect).** The exegesis scroll container reserves a **static `120px`** top inset, but the real header is taller — especially when the message row is expanded ("Tap each highlight (N of M)") and now that the header also adds `env(safe-area-inset-top)` (landed for ticket 12415667946, commit `03e1367`). The read step already solved this by consuming the header's live height via `var(--member-lesson-header)`; the exegesis step never adopted it, so its first line renders under the chrome.
2. **The passage "address"/reference is not shown at all.** The exegesis member step renders only verse rows (number + text) — there is no passage-reference element (e.g. "2 Corinthians 2:3-5") in the template. So even with the inset fixed, no reference/address would appear. Whether one *should* is a small scope question for resolve (the read step carries a `referenceTitle` field; exegesis does not display one).

"Fixed" observably (thread 1): at any header height (collapsed, expanded, notched), the first verse's top line is fully visible below the header, not clipped.

## Root cause
`exegesis-step.vue:607` hardcodes `padding: 120px 16px 24px` on `.ExegesisStep__verse-container`, and the top fade mask (:611-624) hardcodes `120px`. The lesson header is dynamic — its real height (title bar + nav pill + gaps + container/outer padding + **`env(safe-area-inset-top)`** since `03e1367` + the expanded "Tap each highlight" message row) exceeds 120px, so the header overlaps the first verse. `read-step.vue:113` already reserves `calc(var(--member-lesson-header, 200px) + 16px)` from the header's published height and stays clear at any size; exegesis never migrated. Fix: swap the static 120px top inset (and the mask's 120px) for the same `var(--member-lesson-header)`-based calc.
