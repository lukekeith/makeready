# validation error — member SMS verification fails to send

> **Split 2026-07-19:** this ticket originally bundled two unrelated issues. It now tracks ONLY the member web SMS-verification failure. The Jun 20 "changes won't save" report was an unrelated iPhone exegesis-editor bug → moved to **[12572733385](12572733385.md)**.

## Problem
On the member web phone-entry step (`app.makeready.org`, reached in-app from the iPhone app), tapping **Send code** returns red **"Failed to send verification code. Please try again."** — a **Twilio Verify send failure**, not the E.164 phone-format validation error the pre-screenshot triage assumed. Scott hit this both as a leader (Jun 13, via "Open Lesson" routing him into the member funnel — separate issue UX-1 / 12344861586) and again Jun 23; it persists *after* the June SMS-consent fixes because it is a different failure. "Fixed" = the verification SMS actually sends and the member completes the join.

## Root cause
The app code path is correct and defensive; the send throws **inside Twilio Verify** (`twilio.ts:130`). Likely causes are **config/registration**, not a repo bug: misconfigured `TWILIO_VERIFY_SERVICE_ID` (or an org's `twilioVerifyServiceSid`), incomplete A2P 10DLC registration, trial-account/geo restrictions, or carrier filtering. The real reason is captured server-side but was **hidden from the client** by a field mismatch (server returns it under `error`; the join controllers read `message`) — now fixed (Resolution log).
