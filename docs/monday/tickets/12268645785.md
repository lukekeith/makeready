# validation error

## Problem
Two intertwined problems in the study-join flow. (a) A leader tapping "open lesson" from a study enrollment is dropped into the **member** join funnel (phone verification) and then hits a validation error — the leader-preview feature that exists on the server is not wired to this button. "Fixed" looks like: leaders never see phone verification from "open lesson", and members can complete the phone join without an opaque validation failure. The exact error Scott still sees (6/23) is unknown — that's the clarification gap.

## Root cause
Per CF-9 ([code-fixes.md](../code-fixes.md)): `StudyJoinController::showStep` reads `$lesson['organizationId']` at the top level (`StudyJoinController.php:62`) but the API nests it as `lesson.group.organizationId` (`enrollments.ts:2116-2127`) — the null flows into verify-phone/confirm-verification, so members verify without being attached to the study's org. The visible "validation error" is the raw E.164 phone-regex message (`members.ts:420-422`) surfaced by `join-phone-island.vue:104-106`. The leader-side prompt is UX-1: "Open Lesson" resolves to the member join URL instead of the existing leader preview.
