## Root cause
- **Sub-1:** `UnenrollConfirmation.present(...)` hardcodes `style: .warning` for a
  message that is unambiguously a success. Trivial mismatch.
- **Sub-2:** the form uses a full-date `mobile-date` control and nothing enforces a
  minimum age against the group's age range.
- **Subs 3-5:** the reading theme system (blue highlight token, top-aligned themed
  block, full theme picker) is v1-heavy per the reporter; these are design/product
  calls, not defects. Tracked with [[theme-typography-redesign]].
- **Sub-6:** `StudyProgramHome.programsTabContent` never pads its ScrollView bottom,
  so its last card sat under the 106px `ignoresSafeArea(.bottom)` NavBar. That page
  is now unreachable; its live replacement `MainLibrary` pads 100px. NOTE: 100px is
  ~marginally shy of the 106px nav height, so a live re-check on device is warranted
  to confirm the replacement fully clears the frosted nav (and that sibling live list
  pages — MainGroups/MainHome/MainCalendar — carry adequate bottom insets).
