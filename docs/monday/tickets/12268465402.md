# should say lessons

## Problem
On the **Program Home** screen (iPhone modal for a study, e.g. "Struggle Well"), the tab slider reads **Studies | Enrollments | Analytics** — but the first tab's content is the study's *lessons* (day cards: "DAY 1 · test lesson"). A study doesn't contain studies; the label is semantically wrong. The identical label exists on the web LeaderApp's Program Home twin. "Fixed" looks like: the first tab reads **Lessons** on both platforms (and /compare fixtures updated so parity captures still match).

## Reports (verbatim)
- (2026-06-13) "This says 'studies' but should probably say 'lessons'??"

## Root cause
Label semantics, not a defect: the Program Home tab slider names its lessons tab "Studies" on both platforms (`ProgramHomePage.swift:493`, `program-home.vue:202`). Internal naming already says lessons (`ProgramHome__lessons` class, `onReorderLessons`, etc.) — only the user-facing label is wrong.
