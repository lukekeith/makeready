# Release Notes — What Changed Since the Last Archive (2026-06-12)

> Window: since the 2026-06-10 12:09 AM archive — 86 commits. (Sized for TestFlight's 4,000-char field.)

Bible search was rebuilt around meaning, not just words. Searching a concept ("forgiveness," "David and Goliath") now surfaces named passages — pericopes — as first-class results with a title and short summary alongside verse matches, powered by multi-verse window embeddings and LLM expansion of abstract queries; only WEB verses are stored on the device, with other translations served on demand through API.Bible. The result cards were redesigned too: 14-point text with comfortable line height, 16-point padding, no book icon, and the reference stacked beneath passage titles.

The media library took the biggest leap. The full-screen viewer is now a real pager — swipe left or right and the current item rides your finger with its neighbors docked at the edges, the details pane reloading only after the snap. That pane now slides over the media when you drag up, with a properly spaced 64-point title row instead of text hugging its top edge, and the play/pause, fullscreen, and close controls stay visible through playback instead of fading away. Paging is blink-free thanks to a three-image high-res window that keeps memory flat no matter how far you swipe; the redundant play circle is gone from grid video thumbnails (including a long-standing bug where a faint play glyph lingered over cached thumbnails); and big libraries finally scale — no more 100-item cap, cursor pagination that stays fast at hundreds of thousands of items, flash-free grid updates, display-size image decoding with prefetching, and a self-trimming 512 MB image cache.

The app finally tells you when something goes wrong: a red banner slides down from the top whenever an action you just took fails — save, create, delete, upload — usually with a Retry button. Behind it, roughly a hundred previously silent failure points across every screen now route through one error channel — user-initiated failures surface, background refreshes record quietly — with structured, privacy-aware logging replacing raw console prints.

Navigation was consolidated from ten hand-rolled implementations into one system: a single slide container with named motion tokens, real completion handlers instead of timing hacks, and typed routes for overlays, deep links, and cross-tab jumps. Visible bugs died along the way — detail panes popping into place mid-slide (fixed app-wide with a cache-first loading contract), the block-style color picker popping instead of riding its menu — and dead navigation code was deleted outright.

A deep stability and security pass moved the session credential into the Keychain (you stay logged in), re-enabled TLS enforcement, scrubbed sensitive values from logs, and replaced sixteen crash-prone force-unwraps with safe fallbacks. The state layer is now compiler-enforced main-thread-only, a bug that could drop the last save before quitting is fixed, duplicate simultaneous requests share one network call (these had been tripping server rate limits during normal navigation), the authentication manager was slimmed to a single owner of the session credential, the test suite grew from one file to sixty-five tests, and a lint gate keeps the cleaned-up patterns — hardcoded colors and font sizes, console logging, force-unwraps, animation timing hacks — from creeping back in.

For development builds, Local/Staging/Production are now fully isolated — separate sessions and cached data per environment, sign-in re-checked on switch — so data from one server can no longer appear against another. Demo pages and test fixtures no longer ship in release builds.

Please focus testing on: Bible search end to end (concept searches, tapping into the reader), the media viewer (swiping between items, video playback, swipe-down to close, editing details, deep scrolling), creating/editing/deleting program activities and lessons, and the red error banner — try a save in airplane mode and confirm Retry works once you're back online.
